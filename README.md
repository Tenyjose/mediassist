# MediAssist 🏥

An **LLM-native hospital appointment assistant** powered by the Claude API. Patients can view and cancel appointments, authenticate via OTP, and ask natural-language questions about hospital services — in any language — through a conversational chat interface.

## Services

| Service | Port | Description |
|---|---|---|
| `frontend` | 3000 | React chat UI with blue/white hospital branding |
| `conversation-api` | 8002 | LLM-native conversation manager — Claude as the brain |
| `retrieval-api` | 8000 | ChromaDB RAG service with 23 hospital FAQ documents |
| `mock-appointments-api` | 8001 | FastAPI mock hospital system with 20 patient appointments |

---

## Tech Stack

- **LLM**: [Claude API](https://www.anthropic.com/) (`claude-sonnet-4-6`) via the `anthropic` Python library
- **Intent detection**: LLM-native via Claude tool use — no hand-written NLU or Rasa
- **RAG**: [ChromaDB](https://www.trychroma.com/) + [sentence-transformers](https://www.sbert.net/) (`all-MiniLM-L6-v2`)
- **Backend**: [FastAPI](https://fastapi.tiangolo.com/) + [uvicorn](https://www.uvicorn.org/) (Python 3.11)
- **Frontend**: [React 18](https://react.dev/)
- **Auth**: JWT Bearer tokens (simulated OTP flow)
- **Multilingual**: automatic — Claude responds in the patient's language
- **Containerisation**: Docker + Docker Compose

## Quick Start

### Prerequisites
- Docker & Docker Compose
- An Anthropic API key

### 1. Clone and configure

```bash
git clone <your-repo-url>
cd mediassist
cp .env.example .env
# Edit .env and add your ANTHROPIC_API_KEY
```

### 2. Build and run

```bash
docker compose up --build
```

> **Note**: First run downloads the `all-MiniLM-L6-v2` model (~90 MB). Allow ~60 seconds for the retrieval-api to become healthy.

### 3. Open the app

Visit **http://localhost:3000** in your browser.

## API Reference

### Conversation API (`localhost:8002`)

| Method | Path | Description |
|---|---|---|
| `POST` | `/chat` | Send a message; returns `{session_id, response}` |
| `DELETE` | `/session/{session_id}` | Clear session history |
| `GET` | `/health` | Health check |

**Chat request body:**
```json
{
  "session_id": "optional-existing-session-id",
  "message": "Show me my upcoming appointments"
}
```

### Mock Appointments API (`localhost:8001`)

| Method | Path | Description |
|---|---|---|
| `POST` | `/auth/send-code` | Send OTP to email |
| `POST` | `/auth/verify-code` | Verify OTP, receive JWT |
| `GET` | `/appointments?upcoming=1` | List appointments (Bearer token) |
| `POST` | `/cancel-appointment` | Cancel appointment (Bearer token) |

### Retrieval API (`localhost:8000`)

| Method | Path | Description |
|---|---|---|
| `POST` | `/search` | Semantic search over hospital FAQ |
| `POST` | `/answer` | RAG answer generated by Claude |

## Development (without Docker)

```bash
# Terminal 1 — mock appointments
cd mock-appointments-api
pip install -r requirements.txt
uvicorn main:app --port 8001 --reload

# Terminal 2 — retrieval
cd retrieval-api
pip install -r requirements.txt
ANTHROPIC_API_KEY=sk-... uvicorn app.retrieval_api:app --port 8000 --reload

# Terminal 3 — conversation
cd conversation-api
pip install -r requirements.txt
ANTHROPIC_API_KEY=sk-... uvicorn main:app --port 8002 --reload

# Terminal 4 — frontend
cd frontend
npm install
REACT_APP_CONVERSATION_API=http://localhost:8002 npm start
```

## CI/CD

GitHub Actions workflow at `.github/workflows/ci.yml` runs on every push/PR to `main`/`master`:
1. Builds all Docker images
2. Starts all services
3. Health checks each service endpoint
4. Smoke-tests the auth-required `/appointments` endpoint
